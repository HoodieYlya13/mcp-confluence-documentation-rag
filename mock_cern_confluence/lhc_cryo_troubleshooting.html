Operational Notice: LHC Sector 3-4 Cryogenic Interlock

This document details the troubleshooting procedure for the LHC Sector 3-4 vacuum insulation and cryogenic interlocks. Any modification to these thresholds requires a formal approval from the BE-ICS group leader.

LHC Cryo-Vacuum Troubleshooting & Interlock Thresholds

During beam operation, stability of the sub-2K helium bath is critical. Vacuum degradations in the insulating vacuum barrier can cause sudden heat load rises, triggering the cryogenic interlocks.

1. Vacuum Sensor Thresholds

The table below outlines the critical pressure thresholds for the vacuum monitoring system in LHC Sector 3-4.

Sensor ID Location Warning Threshold (mbar) Interlock Trip (mbar) Action Required on Warning
VGPB_34_Q1 Sector 3-4 Q1 Quadrupole 1.2e-5 5.0e-5 Initiate pumping group override, monitor PT-100 temps
VGPB_34_Q2 Sector 3-4 Q2 Quadrupole 1.2e-5 5.0e-5 Initiate pumping group override, monitor PT-100 temps
VGPB_34_D1 Sector 3-4 D1 Dipole 1.5e-5 8.0e-5 Standby for beam dump, isolate sector valves

2. Recovery and Troubleshooting Sequence

  1. If Warning Threshold is breached, check the beam screen heater status via the cryogenic SCADA system.
  2. Execute standard venting validation protocol if a thermal cycle is required.
  3. If Interlock Trip is reached, the beam will dump automatically via the Beam Interlock System (BIS). Verify that energy extraction switches opened successfully.

CRITICAL SAFETY WARNING: Never bypass the BIS input signals for cryo-vacuum. Doing so disables magnet quench protection and may result in catastrophic damage to the superconducting coils.
